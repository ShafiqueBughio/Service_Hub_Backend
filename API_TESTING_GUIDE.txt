================================================================================
                    SERVICELINK API TESTING GUIDE
================================================================================

base_url = http://localhost:3000/api/v1

Note: Auth Required means you need to add header: Authorization: Bearer <access_token>

================================================================================
                            USER ENDPOINTS
================================================================================

1. REGISTER USER
   Method: POST
   URL: {{base_url}}/user/register
   Auth Required: NO
   Body:
   {
     "identifier": "dashti@gmail.com",
     "password": "password@123",
     "user_type": "USER",
     "fcm_token": "optional_fcm_token"
   }
   
   OR
   
   {
     "identifier": "+1234567890",
     "password": "password@123",
     "user_type": "CONTRACTOR",
     "fcm_token": "optional_fcm_token"
   }
   
   Response includes:
   {
     "user": {...},
     "otp": 123456,
     "access_token": "jwt_access_token",
     "refresh_token": "jwt_refresh_token"
   }

--------------------------------------------------------------------------------

2. VERIFY OTP
   Method: POST
   URL: {{base_url}}/user/verify_otp
   Auth Required: YES (use access_token from register)
   Headers: Authorization: Bearer <access_token>
   Body:
   {
     "otp": 123456,
     "fcm_token": "optional_fcm_token"
   }

--------------------------------------------------------------------------------

8. FORGET PASSWORD

   Auth Required: NO
   Body:

   }

--------------------------------------------------------------------------------

9. VERIFY OTP (For Password Reset)
   Method: POST
   URL: {{base_url}}/user/verify_otp
   Auth Required: YES (use access_token from forget_password)
   Headers: Authorization: Bearer <access_token>
   Body:
   {
     "otp": 123456,
     "fcm_token": "optional_fcm_token"
   }
   
   Response:
   {
     "access_token": "jwt_access_token",
     "refresh_token": "jwt_refresh_token",
     "is_profile_completed": true/false
   }
   
   Note: Use this access_token in reset_password

--------------------------------------------------------------------------------

35. GENERATE PRESIGNED UPLOAD URL
    Method: POST
    URL: {{base_url}}/presigned-url/upload-url
    Auth Required: YES
    Body:
    {
      "file_name": "profile.jpg",
      "file_type": "image/jpeg",
      "folder": "contractor-portfolio" (optional),
      "expires_in": 3600 (optional, 60-3600 seconds, default: 3600)
    }
    
    Response:
    {
      "upload_url": "https://s3.amazonaws.com/...",
      "view_url": "https://s3.amazonaws.com/...",
      "file_key": "uploads/1234567890_profile.jpg",
      "file_name": "profile.jpg",
      "expires_in": 3600
    }
    
    Note: Use upload_url to upload file to S3, then use view_url or file_key in your profile

================================================================================
                              NOTES
================================================================================

- Password Requirements:
  * Minimum 8 characters
  * Maximum 16 characters
  * Must contain at least one special character (@$!%*?&)

- OTP:
  * 6-digit numeric code
  * Valid for 60 seconds

- Authentication:
  * Use Bearer token in Authorization header
  * Format: Authorization: Bearer <access_token>

- File Uploads:
  * Use multipart/form-data for file uploads
  * Supported: images and PDFs
  * Max size: 100MB

- Date Format:
  * Use YYYY-MM-DD format (e.g., "1990-01-15")

- User Types:
  * USER: Regular user who can post jobs
  * CONTRACTOR: Service provider
  * ADMIN: Administrator
